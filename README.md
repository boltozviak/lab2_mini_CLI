# Мини оболочка с файловыми командами
## МАИ - Программирование на Python - Лабораторная №2
Реализация маленькой CLI с использованием библиотек Typer и Click_shell

## Команды
- `ls [-l] [-a]` - список файлов
- `pwd` - текущая директория
- `cd <path>` - смена директории
- `cat <file>` - вывести содержимое файла
- `cp <src> <dst> [-r]` - копирование файла/директории
- `mv <src> <dst>` - перемещение/переименование файла/директории
- `rm <path>[-r]` - удаление файла
- `zip <dir> <archive.zip>` - создание zip-архива
- `unzip <archive.zip>` - распаковка zip-архива в текущую директорию
- `tar <dir> <archive.tar.gz>` - создание tar.gz-архива
- `untar <archive.tar.gz>` - распаковка tar.gz-архива в текущую директорию
- `history` - история команд

## Алгоритм работы
1. Устанавливается начальная директория (домашняя)
3. Запускается интерактивный цикл с приглашением `<текущая_директория> >`
4. Пользователь вводит команду с аргументами
5. Typer парсит команду и аргументы
6. Вызывается соответствующая функция из `commands/`
7. Результат выводится в консоль или показывается ошибка
8. Команда сохраняется в историю
9. Цикл повторяется до команды `exit`

## Допущения
1. На шиндоус не работает
2. `cp` и `rm` - требуют --recursive для работы с директориями всегда, даже если они пустые
3. При создании архива обязательного указывать его расширение в названии (.zip / .tar.gz)
4. Есть права для работы со всеми файлами и директориями
